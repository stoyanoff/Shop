# Python Project Rules

## Code Style
- Follow [PEP 8](https://pep8.org/) for code formatting.
- Use meaningful variable and function names.
- Keep lines under 79 characters.

## Structure
- Organize code into modules and packages.
- Place tests in a separate `tests/` directory.

## Dependencies
- List all dependencies in `requirements.txt`.
- Use virtual environments for development.

## Version Control
- Commit code regularly with clear messages.
- Do not commit sensitive information.

## Documentation
- Document all public functions and classes.
- Update `README.md` with setup and usage instructions.

## Testing
- Write unit tests for all major functionality.
- Ensure all tests pass before merging code.

## Code Reviews
- Submit pull requests for all changes.
- Address review comments promptly.
